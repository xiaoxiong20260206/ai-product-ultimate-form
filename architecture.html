统一AI产品架构全景图

三层架构 · 四类技能 · 六大能力 · 成长闭环
一页讲清整个产品矩阵的技术根基

1

三层架构总览

整个AI产品技术栈分为三层。产品层面向用户，决定"长什么样"；Agent层是智能中枢，决定"怎么做事"；AgentOS层是能力基座，决定"能做什么"。三层解耦，上层可独立换壳，下层统一复用。

2

产品层：三个产品共享一个引擎

三个产品线面向不同用户群体，但底层共享同一个 Agent 引擎。差异化来自产品适配层——不同的 UI 壳、不同的 Skill 子集、不同的权限模型。这意味着新能力（Skill）只需开发一次，即可按需开放给多个产品。

🔌 产品适配层（差异化关键）

同一个 Agent 引擎通过适配层为不同产品"换壳"，实现"一次构建，N 次复用"

UI 适配

同一能力在 IDE / Web / 移动端呈现不同组件和交互方式

Skill 筛选

按产品定位过滤可用 Skill，防止"全量能力暴露"导致的体验混乱

权限模型

租户 / 产品 / 用户三级隔离；企业场景需要更严格的审计和开关

Ⅰ 原生 Skill

Native

判定标准
主要靠文本/结构化推理即可完成，不依赖外部 API，无需并行拆分
执行方式
主 Agent 读取 SKILL.md 后直接执行（可被自然语言"教会"）
失败模式
信息源不足 / 指令不够具体 / 需要外部动作却没工具
📊 行业调研 📝 文档撰写 📑 PPT制作 🔍 深度分析
Ⅱ 工具 Skill

Tool / MCP

判定标准
需要调用外部服务一次或少量次即可完成（生成 / 查询 / 执行）
执行方式
主 Agent → MCP 网关 → 外部 API；网关统一鉴权 / 限流 / 路由
失败模式
鉴权失败 / 限流超时 / 外部服务波动
🖼️ 图片生成 🎬 视频生成 🔄 CI/CD 📈 数据查询
Ⅲ 任务 Skill

Task / 分身并行

判定标准
可拆成多个相互独立的子任务并行（多视角 / 多维度 / 多分片）
执行方式
分身引擎派生 N 个实例；共享 Memory 快照、独立上下文；结果汇总 + 经验回流
失败模式
拆分不合理 / 汇总冲突 / 并发资源不足
🔎 CR代码审查 ♻️ 重构评审 📐 多维需求分析
Ⅳ 流程 Skill

Flow / 阶段编排

判定标准
存在多阶段依赖链（Stage DAG），且需要过程可见、可中断、可重试
执行方式
两层编排：Flow 层（阶段）+ Task 层（阶段内执行任意 Skill）
失败模式
阶段依赖不清 / 数据传递缺失 / 断点恢复不完整
🚀 需求→发布全链路 🏗️ 项目初始化 📦 批量迁移

🧭 Skill 类型快速判断

只要写/想？
→ 先用 Native（最轻量）
要调 API？
→ 用 Tool（走 MCP 网关）
可并行拆？
→ 用 Task（分身并行）
有阶段依赖？
→ 用 Flow（阶段编排 + 断点）
4

AgentCore：六大能力的内部关系

AgentCore 是 Agent 的"大脑"，由六大能力模块组成。它们并非平等并列的关系——Memory 和 Integration 是底层基础设施，Reasoning → Planning → Orchestration 构成主处理链，Reflection 形成反馈闭环。

🧠 Memory

长期记忆 · 护城河
职责
读写记忆；去重/提炼；可审计可删除
分层
L1工作记忆 / L2情景记忆 / L3语义记忆
隔离
租户→产品→用户；敏感可删除

🔍 Reasoning

理解 · 推断
职责
意图识别 / 消歧 / 隐含需求推断
输入
用户输入 + Memory 上下文
输出
结构化意图（Intent）与约束

📋 Planning

规划 · 分解
职责
把 Intent 分解为可执行的 Stage DAG
输入
Intent + 工具/Skill 能力清单
输出
计划（步骤 / 依赖 / 校验点 / 回滚点）

⚙️ Orchestration

调度 · 执行
职责
并发调度（超时/重试/资源控制）
输入
Planning 产出的 DAG + Runtime 约束
输出
执行轨迹与中间态（供观测/回放）

🔄 Reflection

反思 · 进化
职责
自评/复盘；把经验写回 Memory
输入
结果 + Eval 分数/错误原因
输出
改进建议 / 经验条目 / 策略调整

🔌 Integration

外部接入 · 路由
职责
外部服务接入与路由（LLM/搜索/CI）
输入
调用请求 + 鉴权/配额
输出
统一结果/错误码/重试语义

7

成长闭环：Agent 越用越强

与传统软件不同，基于这套架构的 Agent 会成长。每次执行任务后，Reflection 模块自动评估质量、提炼经验、写入 Memory。下次面对类似任务时，Agent 会调用这些经验做得更好。这是一个持续的正反馈循环。

执行 → 反思 → 记忆 → 进化

执行任务
→
Reflection 自评
→
经验写入 Memory
→
下次执行更好
↩

L1 行为适应

记住偏好、习惯、上下文
→ 减少重复沟通

L2 技能进化

从执行中提炼方法论
→ Agent 写 Skill 给自己

L3 认知进化

建立跨领域知识图谱
→ 举一反三、触类旁通
