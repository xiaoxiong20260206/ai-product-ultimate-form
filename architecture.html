⚡ 四类技能体系详解（hover 看细节）

Ⅰ 原生 Skill (Native)

会话模式：派 1 个独立会话，带 Skill 指令独立完成；不依赖外部 API

游戏类比：🎮 派一个角色去做单人支线任务，做完回来交经验

生命周期：创建 → 注入Skill → 独立执行 → 经验回流 → 会话销毁

📊 行业调研 📝 文档撰写 📑 PPT制作 🔍 深度分析

Ⅱ 工具 Skill (Tool/MCP)

会话模式：派 1 个独立会话，通过 MCP 网关调用外部 API；网关统一鉴权/限流

游戏类比：🎮 派一个角色去 NPC 商店买装备/找铁匠打造

生命周期：创建 → 注入Skill → 调API → 经验回流 → 会话销毁

🖼️ 图片生成 🎬 视频生成 🔄 CI/CD 📈 数据查询

Ⅲ 任务 Skill (Task)

会话模式：派 N 个并行会话，各带不同 Skill；各自独立上下文，结果汇总后经验回流

游戏类比：🎮 组队打副本：多角色同时出战，各有分工

生命周期：创廾N个 → 并行执行 → 汇总 → 经验回流 → 全部销毁

🔎 CR代码审查 ♻️ 重构评审 📐 多维需求分析

Ⅳ 流程 Skill (Flow)

会话模式：1 个总控会话编排多阶段，每阶段派执行会话（可是Native/Tool/Task任意组合）

游戏类比：🎮 多阶段 Raid Boss 战：指挥官喜户多小队，各司其职

能力：跳过/重试/断点续跑/人工确认门禁；总控会话最后销毁

🚀 需求→发布全链路 🏗️ 项目初始化 📦 批量迁移
🧭 Skill 类型决策指南（P1）
快速判断（按会话复杂度）
1个会话够？
→ 用 Native（不调外部）
要调外部API？
→ 用 Tool（1个会话+MCP网关）
可并行拆？
→ 用 Task（N个并行会话协作）
有阶段依赖？
→ 用 Flow（1总控+N会话编排）
建议产品侧在PRD里写清：触发词、输入约束、输出形态、失败兜底；技术侧写清：依赖、权限、SLA、回滚。
🧬 Skill 生命周期与治理（P1/P2）
开发 → 注册(元数据/版本) → 灰度/AB → 评估(Eval) → 发布 → 回滚/下线
关键治理项：Schema校验、权限审计、成本预算(Token/调用)、SLA、观测面板、风险开关。
🧠 Memory 分层与隔离策略（P0）
L1 工作记忆
会话上下文/临时变量
特点：低延迟；自动过期；不做长期沉淀
L2 情景记忆
偏好/历史/任务记录
特点：可审计；可删除；支持时间衰减
L3 语义记忆
知识/文档/经验索引
特点：语义检索；去重聚合；引用溯源
隔离：租户（企业/个人）→ 产品（IDE/Web/助手）→ 用户；共享仅限“通用偏好/匿名经验”，敏感信息必须可删除、可审计。

🎮 如果 Agent 架构是一个游戏世界

左侧是产品 & 技术架构，右侧是网络游戏类比 — 逐层映射，秒懂每一个抽象概念

📦 产品层 — 不同产品形态

同一套 Agent 内核，通过产品适配层（UI · Skill集 · 权限）打造出完全不同的产品：

CodeFlicker 本地IDE插件，需下载安装
联网调用LLM，低延迟优先
KATE-Studio 纯Web应用，零安装
企业统一管控，批量任务
AI个人助理 多端：本地+Web+移动
云端为主，全量能力开放
⟺

🎮 不同游戏 — 同一引擎开发

同一套游戏引擎，开发出完全不同类型的游戏（不是同一游戏的不同职业）：

🖥️ 本地客户端游戏 需要下载安装客户端
联网但本地运算优先
🌐 网页游戏 打开浏览器即玩
轻量、即开即用
🌍 大型MMO网游 客户端+网页+手机多端
云端服务器为核心

🧠 Agent层 — 决策 + 会话派遣

主 Agent 永不下场执行。它理解意图、查记忆、选Skill，然后派出独立会话（分身）去执行。

会话 = 独立上下文，执行完毕后经验回流、会话销毁。

Ⅰ Native / Ⅱ Tool 1个独立会话，带Skill完成
Tool多一步调外部API
Ⅲ Task / Ⅳ Flow Task: N个并行会话协作
Flow: 1总控 + N执行会话
⟺

🎯 玩家 + 技能系统

玩家自己不进入游戏世界，通过召唤角色/分身去执行任务。任务完成后角色消失，经验归属玩家。

每次召唤 = 一个独立游戏实例。

🗡️ 单人任务 / NPC交互 派一个角色独立完成
NPC交互=调用外部服务
⚔️ 组队副本 / Raid 多角色同时出战协作
Raid=指挥官+多小队编排

⚙️ Agent OS — 底层运行时

提供所有会话共用的基础能力，与Agent层在同一进程内（函数调用，非远程通信）。

  • AgentCore 六能力：Memory · Reasoning · Planning · Orchestration · Reflection · Integration
  • Session Manager：创建/监控/回收/汇总独立会话
  • MCP网关：统一外部API调用入口
⟺

🏗️ 游戏引擎

所有游戏共用的底层引擎（如Unreal/Unity），与游戏逻辑在同一进程内运行。

  • 引擎核心：存档系统 · AI行为树 · 任务系统 · 调度器 · 回放/复盘 · 网络层
  • 副本系统：创建/管理/回收独立游戏实例
  • 插件API：统一外部Mod和服务接入

🔄 成长闭环 — 越用越强

执行任务 → Reflection自评 → 经验写入Memory → 下次执行起点更高。

会话用完即毁，Memory持久存活。这是Agent区别于传统软件的本质。

L1 行为适应记住偏好和习惯
L2 技能进化从执行中提炼方法论
L3 认知进化跨领域知识图谱
⟺

⭐ 经验值 + 升级系统

打怪获得经验 → 升级解锁新技能 → 更强角色打更难副本。

角色消失但经验永存。这是游戏让人上瘾的核心机制。

L1 熟练度操作越来越熟练
L2 天赋树解锁新技能和流派
L3 跨服成就跨游戏的通用认知

💡 核心洞察：分身 = 独立会话（Session）

主Agent派出的每个"分身"本质上是一个独立的会话实例。会话有自己的上下文、按Skill指令执行、完成后将经验反馈回主Agent然后销毁。
任务的复杂度差异，本质只是需要多少个会话、如何协调的区别。
